# Cascade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Normative spec:** `docs/cascade-game-scope.md` — every executor MUST read it. Where this plan and the spec disagree, **the spec wins**. The spec contains the exact algorithms, scoring formulas, and test values; this plan locks in the file structure, module contracts, and build order so parallel workers cannot drift.

**Goal:** Build and fully verify Cascade — an offline, portrait, drag-and-drop 8×8 block puzzle with gravity cascades — per the spec's Definition of Done (§7.6), deployable as a PWA playable on Android and iPhone.

**Architecture:** A pure, renderer-independent rules engine (`src/engine`, zero DOM imports, all randomness through one injectable seeded RNG) consumed by (a) a headless simulation harness, (b) a canvas renderer + DOM-overlay UI, and (c) a pure `MonetizationDirector`. Persistence goes through a `Storage` interface. Every animation is driven by the `ChainStep[]` data the engine returns — the renderer never re-derives game logic.

**Tech Stack:** TypeScript (strict), Vite, HTML5 Canvas 2D, Web Audio API, Vitest (unit + sim), Playwright (e2e), vite-plugin-pwa (service worker + manifest), GitHub Pages (hosting).

## Engine API contract (normative for all workers)

```ts
// src/engine/types.ts
export type Cell = number;                       // 0 = empty, 1..8 = color index (cosmetic only)
export type Board = Cell[];                      // length 64, index = row * 8 + col
export const COLS = 8, ROWS = 8;
export const idx = (col: number, row: number) => row * 8 + col;

export interface PieceDef { id: string; cells: ReadonlyArray<readonly [number, number]>; w: number; h: number; }
export interface TraySlot { pieceId: string; color: number; }

export interface FallMove { col: number; fromRow: number; toRow: number; color: number; }
export interface ChainStep {
  step: number;                                  // 1-based chain counter
  rowsCleared: number[]; colsCleared: number[];
  clearedCells: Array<{ col: number; row: number; color: number }>;  // each cell once (cross counted once)
  linesCleared: number;                          // rows + cols this step
  basePoints: number;                            // linesCleared² × 10
  pointsAfterChain: number;                      // basePoints × step
  fallMoves: FallMove[];                         // gravity moves AFTER this clear
  boardAfter: Board;                             // snapshot after clear+gravity (for renderer & tests)
}
export interface PlacementResult {
  placementPoints: number;                       // = piece cell count
  steps: ChainStep[];                            // empty if no clear (then NO gravity ran)
  streakAfter: number;
  streakMultiplier: number;                      // min(streakAfter,5) if streakAfter ≥ 2 else 1
  linePoints: number;                            // Σ pointsAfterChain × streakMultiplier
  allClearBonus: number;                         // 300 or 0
  totalPoints: number;                           // placementPoints + linePoints + allClearBonus
  scoreAfter: number;
  maxChain: number;                              // run-level max after this placement
  trayRefilled: boolean;
  gameOver: boolean;
}
export interface ContinueResult { steps: ChainStep[]; boardAfter: Board; }  // ZERO points (§9.3.1)

export type GameMode = 'normal' | 'daily';
export interface GameState {
  board: Board; tray: (TraySlot | null)[];       // tray length always 3; null = consumed
  score: number; streak: number; maxChain: number;
  placements: number; continueUsed: boolean;
  rngState: number; mode: GameMode; dailyDate: string | null;  // 'YYYYMMDD'
  over: boolean;
}
```

**Pinned rule interpretations** (the spec leaves these implicit; pinned here so tests and impl agree):
1. **Streak & multiplier:** streak increments when a placement triggers ≥1 clear (any chain step) and resets to 0 otherwise. The multiplier `min(streak, 5)` (when streak ≥ 2) uses the streak **including the current placement** — i.e. 2nd consecutive clearing placement gets ×2, 3rd gets ×3.
2. **Cascade chain counter** starts at 1 for the placement-triggered clear (§2.4.4).
3. **Gravity runs only after a clear** (§2.4 note) — a no-clear placement produces `steps: []` and no gravity.
4. **Game-over check** (§2.7) considers only non-null tray slots; runs after refill if the tray was refilled.
5. **Continue reward** (§9.3.1): clear the 2 rows with the most filled cells (tie → lower row index); then the normal gravity/cascade loop runs (chain counter starts at 1); **zero points from any of it** (no line points, no all-clear bonus) and **no stat changes** (streak and maxChain untouched — the reward is survival only). `continueUsed := true`.
6. **Tray colors** are cosmetic but flow through the seeded RNG so daily runs look identical for everyone.
7. **Daily seed:** `hashStringToSeed('YYYYMMDD')` (FNV-1a 32-bit) of the **local** date.
8. **Refill order** (§2.4.6 vs §2.7): score → refill if all 3 consumed → game-over check.
